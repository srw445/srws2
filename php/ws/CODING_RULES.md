# ws コーディング規約

## 1. ファイル・ディレクトリ構成
- MVCパターンを基本とし、`controllers`・`models`・`views`・`sql`・`ddl`等で整理する
- SQLは`sql/`、DDLは`ddl/`に分離
- 設定・機密情報は`config/`にまとめ、git管理外とする

## 2. 命名規則
- クラス名: パスカルケース（例: `VideoController`）
- メソッド・変数名: キャメルケース（例: `getVideoRecords`）
- ファイル名: 英小文字+アンダースコア（例: `video_record.php`）
- テーブル・カラム名: 日本語（全角）を許容、ただし一貫性を重視
- SQLファイル: 機能名+用途（例: `get_video_records.sql`）

## 3. コーディングスタイル
- インデントはスペース4つ（PHP, SQL, JS, CSS等）
- PHPの`<?php ... ?>`は省略せず明示
- 1行80文字以内を目安
- コメントは日本語で簡潔に
- 関数・クラスの冒頭に目的コメントを記載

## 4. セキュリティ
- SQLは必ずプリペアドステートメントを利用
- セッション管理は`session_start()`の多重呼び出しを避ける
- 入力値は`htmlspecialchars`等でサニタイズ
- パスワード等は平文保存禁止

## 5. Git運用
- `config/`や`vendor/`等は`.gitignore`で除外
- コミットメッセージは「何を・なぜ」日本語で簡潔に
- mainブランチは常に動作する状態を保つ

## 6. その他
- Bootstrap等の外部ライブラリは`vendor/`配下に配置
- ライセンスファイルは必ず同梱
- 画面設計はユーザビリティ・アクセシビリティを重視

---

本規約はプロジェクトの進行に応じて随時見直し・追記する。
